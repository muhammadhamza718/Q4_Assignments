# Project Task List: Car Rental Website

This document provides a detailed, granular checklist of all tasks required to complete the car rental website project using HTML, CSS, and JavaScript.

## Phase 1: Project Setup & Configuration

- [x] Create the project directory structure (`css/`, `js/`, `images/`, `assets/`).
- [x] Create core files: `index.html`, `css/style.css`, `js/script.js`.
- [x] Add CDN links for Boxicons and Google Fonts in `index.html`.

## Phase 2: HTML Structure Development

- [x] Create the basic HTML5 boilerplate structure.
- [x] Implement the `Header` section (logo, search bar, user icons).
- [x] Implement the `Hero` section (promotional cards).
- [x] Implement the `Search` form section (pick-up/drop-off details).
- [x] Implement the `CarList` sections with `CarCard` components.
- [x] Implement the `Footer` section (company info, navigation, legal links).
- [x] Ensure the HTML structure is semantic and accessible.

## Phase 3: CSS Styling

- [x] Define CSS custom properties (variables) for colors, fonts, and spacing.
- [x] Add global CSS resets and base styles.
- [x] Style the `Header` section.
- [x] Style the `Hero` section.
- [x] Style the `Search` form section.
- [x] Style the `CarCard` components.
- [x] Style the `CarList` sections.
- [x] Style the `Footer` section.
- [x] Ensure responsive design for mobile and tablet viewports.
- [x] Add utility classes (e.g., `.hidden`).

## Phase 4: JavaScript Interactivity

- [x] Implement the "Like" button functionality (toggle icon state).
- [x] Implement the "Swap" button functionality (exchange pick-up/drop-off values).
- [x] Implement the "Show More" functionality (reveal hidden car cards).
- [ ] Add form validation for the search form.
- [ ] Consider adding smooth scrolling or animations for better UX.

## Phase 5: Testing & Refinement

- [ ] Test all interactive features (like, swap, show more).
- [ ] Perform a responsive design audit on mobile and tablet viewports.
- [ ] Perform a final code review for clarity and best practices.
- [ ] Check for and fix any accessibility issues.
- [ ] Validate HTML and CSS code.

## Phase 6: Deployment

- [ ] Choose a static hosting service (GitHub Pages, Netlify, Vercel, etc.).
- [ ] Upload all project files to the hosting service.
- [ ] Configure the hosting service to serve `index.html` as the main page.
- [ ] Test the deployed website to ensure all features work correctly.
